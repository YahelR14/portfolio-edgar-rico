# Portfolio Web — Edgar Rico

## Objective

Build a professional personal portfolio for a Software Engineering student.

The website must demonstrate technical skills, software engineering practices,
UX/UI awareness, and software quality experience through real project evidence.

## Target audience

* Recruiters
* Hiring managers
* Software engineering teams
* Reviewers for trainee and internship opportunities

## Stack

* Astro
* TypeScript
* Modern CSS
* Astro content collections
* Static deployment with Vercel

## Required sections

* Sticky header
* Hero
* About me
* Technical skills
* Featured projects
* Education
* Downloadable resume
* Contact
* Footer

## Featured projects

1. CoffeeRank TJ
2. SauceDemo QA Automation
3. CIMA

## Design rules

* Prioritize a professional, modern, restrained, and intentional visual style.
* Avoid the appearance of a generic developer portfolio template.
* Use full-width sections with readable inner content widths.
* Do not wrap the entire website inside one large boxed container.
* Use cards selectively and only when they improve information grouping.
* Use a compact sticky header.
* Keep animations subtle and purposeful.
* Respect `prefers-reduced-motion`.
* Prioritize clear hierarchy, typography, spacing, and real project evidence.
* Ensure the interface works correctly on mobile and desktop.

## Engineering rules

* Use semantic HTML.
* Maintain visible keyboard focus states.
* Avoid unnecessary client-side JavaScript.
* Avoid unnecessary dependencies.
* Optimize project images.
* Use Astro content collections for project information.
* Run `npm run build` after significant changes.
* Review the final diff before considering a task complete.

## Git rules

* Do not run `git push` without explicit approval.
* After a significant change, summarize the modified files.
* Ask whether to commit and push before proceeding.
